## Sobre o Projeto

Este repositório concentra a documentação técnica e os artefatos de engenharia do **braço robótico** desenvolvido pelo time **Robotics Titans**, vinculado à **Universidade de Brasília (UnB)**.

### Objetivos

O projeto visa estruturar, de forma rastreável e colaborativa, o desenvolvimento multidisciplinar do sistema. Os principais objetivos são:

- **Documentação unificada**: Centralizar especificações, decisões de projeto e registros técnicos em um portal MkDocs
- **Integração entre áreas**: Explicitar interfaces entre Software, Controle, Mecânica e Eletrônica
- **Rastreabilidade**: Registrar requisitos, status e entregáveis de cada disciplina
- **Padronização**: Manter um padrão editorial corporativo para engenharia (Markdown, tabelas, status e versionamento)

## Documentação

**Site de Documentação**: [https://team-titans-unb.github.io/bracoRobotico/](https://team-titans-unb.github.io/bracoRobotico/)

A documentação completa inclui:

- Portal institucional do projeto
- Páginas por área técnica (Software, Controle, Mecânica e Eletrônica)
- Visões gerais e próximos registros de cada disciplina
- Guia Markdown com o padrão editorial da equipe

## Estrutura do Projeto

```text
bracoRobotico/
├── docs/                         # Conteúdo da documentação (Markdown)
│   ├── index.md                  # Página inicial
│   ├── guia-markdown.md          # Padrão editorial / cheatsheet
│   ├── software/                 # Área de Software
│   ├── controle/                 # Área de Controle
│   ├── mecanica/                 # Área de Mecânica
│   ├── eletronica/               # Área de Eletrônica
│   └── stylesheets/              # CSS corporativo
├── overrides/                    # Customizações do tema Material
│   └── partials/
├── .github/
│   └── workflows/
│       └── deploy.yml            # Deploy automático no GitHub Pages
├── mkdocs.yml                    # Configuração do MkDocs
├── requirements.txt              # Dependências Python
└── README.md
```

## Contribuição

1. Atualize a documentação na área correspondente (`docs/software`, `docs/controle`, `docs/mecanica` ou `docs/eletronica`)
2. Siga o padrão editorial em [`docs/guia-markdown.md`](docs/guia-markdown.md)
3. Valide localmente com `python -m mkdocs serve`
4. Faça push na `main` — o deploy no GitHub Pages ocorre automaticamente via Actions
